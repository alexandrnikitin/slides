Part 2: One Million RPS

Attribution request sample

  • Request: http://xd.root.adform.com:8080/adform/uniqueId?cid=21231231231
  • Response: {"auid":"8593283872557236166"} OR {"code":404,"message":"Unique not found."}

Numbers

  • 1 min run, 33% hit: 1113 KTPS with, ~1ms avg, 4.36-4.47ms 99 percentile,
  • 20 min run,16% hit: 1072 KTPS with, ~2.2ms avg, 1.2ms 90s percentile, but up to 58ms 99th percentile

Hardware

  • 4 Application nodes in 2 DCs, 32core + 256GB RAM each + 2 network cards (6 ports in total)
  • 4 Aerospike nodes in 2 DCs, 32core + 256GB RAM + 5 SSD each

How?

CQRS

  • Separate read only endpoint.
  • Single Aerospike read per request.

Tech stack:

  • Scala
  • Netty
  • Aerospike

Netty (web app)

  • Works well out of the box
  • Tuned params: worker threads
  • Concept of buffers (ByteBuf)
  • For keep-alive connections: flush on channelReadComplete only
  • No-framework JSON serialization

Aerospike DB

  • 4 nodes in 2 Datacenters
  • Rack-aware configuration
  • Storage engine is "device": data is on SSDs
  • Client setting: maxThreads = 30000
  • Client could be better (CPU inefficiency)
